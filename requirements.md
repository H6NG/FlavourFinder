Requirements

Login and accounts:
  Login page exists
  User can login (can be a guest).
  User can logout.
  User can sign up for an account.
  User preferences/dietary restrictions are preserved.
  
Finding a restaurant
  Will give you a recommendation
  Follows dietary restrictions
  Recommendation is within the distance specified
  Displays the restaurant details (name and address)
  Able to copy address
  Displays image of restaurant (if available)
  Able to reroll
  Negatively affects the rerolled restaurant’s ranking


Preferences
  Users can input their dietary preferences
  Users can input their current location
  Users can have their location automatically detected (“Allow x to access your location?)
  Users can select a search radius (distance)
  Can input a location or allow the permission to get their location through the browser
  Suggestion will follow preference


Rating System:
  Prompts user to rank 3 restaurants after each meal
  One of them is the one previous 
  One of them is the previous previous
  The other is randomly selected from the user’s previous restaurants or possible based on rankings to break ties
  Saves the user choice to database
  Ranks restaurant based on user choice.
  User cannot see their ratings
